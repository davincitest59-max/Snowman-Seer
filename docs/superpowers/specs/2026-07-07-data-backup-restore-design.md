# 数据备份与恢复设计

## 目标

为 Ocean Baby 增加“一键导出全部数据”和“新安装后导入恢复全部数据”的能力。用户可以把本机数据导出成一个文件，之后在同一台手机重装应用或换到另一台安卓手机后，通过导入这个文件恢复账本、笔记、笔记图片、待办、心情记录和设置。

本设计采用已确认的方案二：导出为一个 `.oceanbaby` 单文件，文件内部使用 JSON 保存结构化数据，并把笔记图片作为文件一起打包。

## 范围

本次功能只面向安卓端本地数据备份与恢复，不增加登录、云同步、电脑端、浏览器端或 iOS 端能力。

导入模式固定为“覆盖恢复”：导入前提示用户确认，确认后清空当前 Ocean Baby 本地数据，再写入备份文件中的数据。该模式适合新安装、卸载重装和换机恢复，避免合并导入带来的重复账单、重复笔记和图片冲突。

## 用户入口

在“设置”页新增一个可展开、可收起的分类：

- 数据备份与恢复

分类中包含两个操作：

- 导出全部数据
- 导入恢复数据

导出成功后显示提示，说明备份文件已生成，并引导用户保存或分享该文件。导入时先让用户选择 `.oceanbaby` 文件，再显示二次确认提示：导入会覆盖当前全部本地数据，请确认已经保存好当前数据。

## 备份文件格式

导出的文件扩展名为 `.oceanbaby`，文件名格式建议为：

`OceanBaby_yyyyMMdd_HHmmss.oceanbaby`

该文件本质是一个压缩包，内部结构如下：

```text
manifest.json
data.json
note_images/
```

`manifest.json` 保存备份元信息：

- appName：固定为 Ocean Baby
- backupFormatVersion：备份格式版本，初始为 1
- databaseSchemaVersion：当前数据库结构版本
- exportedAt：导出时间
- platform：导出平台

`data.json` 保存全部结构化数据：

- ledgerRecords：账本记录
- notes：笔记记录
- todos：待办记录
- moods：心情历史记录
- settings：设置项

`note_images/` 保存笔记图片文件。`data.json` 中的笔记图片字段不保存旧手机的绝对路径，而保存备份包内的相对文件名列表。导入后，应用把图片复制到当前手机的应用图片目录，再把笔记中的图片路径改为新手机上的实际路径。

## 数据导出流程

用户点击“导出全部数据”后：

1. 应用读取 SQLite 中的账本、笔记、待办、心情和设置表。
2. 应用整理成稳定的 JSON 结构，不直接导出旧数据库文件。
3. 应用扫描笔记中的图片路径，把存在的图片复制进备份包的 `note_images/` 目录。
4. 如果某张图片文件已经不存在，仍然导出笔记文字数据，但在导出结果中提示有图片缺失。
5. 应用生成 `.oceanbaby` 文件，并让用户保存或分享该文件。

导出过程不修改当前数据库。

## 数据导入流程

用户点击“导入恢复数据”并选择备份文件后：

1. 应用校验文件是否为 Ocean Baby 备份文件。
2. 应用读取 `manifest.json`，确认备份格式版本可支持。
3. 应用读取 `data.json`，校验必要字段存在。
4. 应用显示覆盖恢复确认弹框。
5. 用户确认后，在一个事务中清空当前账本、笔记、待办、心情和设置数据。
6. 应用把备份包中的图片复制到当前手机应用文档目录下的 `note_images` 文件夹。
7. 应用写入账本、笔记、待办、心情和设置数据。
8. 应用刷新当前页面状态，并提示“数据已恢复”。

如果导入失败，应用应尽量保持当前数据不被破坏。清空旧数据和写入新数据需要放在同一个数据库事务里；图片复制失败时，应取消导入并提示用户重新选择备份文件。

## 兼容性

备份格式使用独立的 `backupFormatVersion`，不直接绑定 SQLite 文件格式。这样后续数据库字段变化时，可以在导入服务中做格式升级。

首版只要求支持当前版本导出的备份文件。遇到更高版本的备份文件时，应用提示“备份文件版本过高，请升级 Ocean Baby 后再导入”。

## 错误处理

需要处理以下情况：

- 用户取消导出或导入。
- 选择的文件不是 Ocean Baby 备份文件。
- 备份文件损坏或缺少 `manifest.json`、`data.json`。
- 备份文件版本不支持。
- 备份内图片缺失或图片复制失败。
- 手机存储空间不足。

所有错误提示均使用中文，避免出现英文系统文案。

## 测试要求

需要覆盖以下测试：

- 导出文件包含 `manifest.json`、`data.json` 和笔记图片文件。
- 导出的 JSON 包含账本、笔记、待办、心情和设置。
- 导入后数据数量和关键字段与导出前一致。
- 导入时会覆盖当前旧数据，而不是合并旧数据。
- 笔记图片导入后路径指向当前手机新的应用目录。
- 非 Ocean Baby 文件导入失败并显示中文提示。
- 设置页显示“数据备份与恢复”分类，并能触发导出和导入入口。

## 不做内容

本次不做云同步，不做自动上传，不做账号登录，不做 QQ 或微信登录绑定，不做隐藏式数据查看，也不尝试读取其他应用的内部数据库。
